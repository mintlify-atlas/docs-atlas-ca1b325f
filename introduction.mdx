DOSBox Staging is a modern continuation of the original DOSBox project, built to run MS-DOS software — games, utilities, and applications — on current Windows, macOS, and Linux systems. Unlike a full-system emulator such as PCem or 86Box, you do not need to install an operating system image or configure virtual hardware from scratch. DOSBox Staging provides a ready-to-use MS-DOS environment the moment you launch it, with built-in emulation of the audio and video hardware that DOS software expects.

## What DOSBox Staging emulates

DOSBox Staging covers the breadth of the IBM PC compatible era — from the early 1980s XT machines through Pentium-era systems of the late 1990s. It also handles the Tandy 1000 and IBM PCjr clones, and can run early self-booting "PC booter" games that predate DOS entirely.

## How DOSBox Staging differs from vanilla DOSBox

The original DOSBox project slowed significantly after 2010, leading to a fragmented ecosystem of forks and custom builds. DOSBox Staging was started in 2020 by Patryk "dreamer" Obara and kcgen to reunify those efforts under a single, actively maintained codebase. It received the blessing of original DOSBox co-author Sjoerd "Harekiet" van der Berg.

## DOSBox Staging vs. full PC emulators

Full system emulators such as PCem and 86Box reproduce an entire IBM PC compatible machine at the hardware level. You select a motherboard, CPU, video card, and sound card, then install MS-DOS from floppy images before running any software. This gives you maximum hardware fidelity and the ability to run operating systems other than DOS, but it also means significant upfront setup work for every configuration you want to maintain.

DOSBox Staging takes a different approach: it emulates only the MS-DOS environment and the hardware components that DOS software actively uses. Games start immediately without a boot sequence. Changing a sound card is a one-line config edit rather than a virtual machine rebuild. This makes DOSBox Staging substantially easier to use for maintaining a large game library.

<Note>
  If you need cycle-exact hardware accuracy for preservation research or software development, a full PC emulator is the better fit. DOSBox Staging optimises for convenience and playability — with the requirement that supported software runs *at least* as well as on real period hardware.
</Note>

## A note for users of other DOSBox variants

If you have previously used DOSBox SVN, DOSBox-X, or another fork, you will find that DOSBox Staging behaves similarly for most games. Configuration key names are largely compatible. The main differences are in defaults (the CRT shader, the aspect ratio handling, and the mixer) and in the location of the primary configuration file. See the [installation guide](/installation) for the path to the primary config on each platform.
